最終更新: 2026-06-27

## 1. はじめに

### 1.1 本書の目的

本書は、ToDoリストカレンダーアプリ **TiDepo** の要件を定義する。プロダクトの目的、対象範囲、機能要件、非機能要件、技術・アーキテクチャ方針、外部連携、データモデル、コア計算ロジック（猶予時間）、制約、将来構想を整理し、設計・実装フェーズの起点とすることを目的とする。

本書は要件レベルの記述にとどめ、具体的な製品選定（認証基盤・データストア等）や詳細設計は、後続の設計フェーズで決定する（[10. 技術・アーキテクチャ方針](#10-技術アーキテクチャ方針) 参照）。

### 1.2 背景

- 本プロジェクトは、趣味・個人開発として行うToDoリストカレンダーアプリである。
- 同等のプロダクトを Next.js / React / TypeScript / Firebase 構成で一度実装済み（参考リポジトリ `todolistcalendar2`）であり、機能要件・コアロジックはそこで検証済みである。
- 今回はアーキテクチャと技術スタックを見直すため、ゼロから作り直す。プロダクトの価値・機能は踏襲しつつ、バックエンドを Java / Spring Boot、フロントエンドを TypeScript / React で再構築する。

### 1.3 プロダクトの一言要約

TiDepo は、予定とタスクを同じカレンダー上で管理し、締切までに **「本当に残っている作業可能時間（猶予時間）」** を可視化する個人向けタスク管理アプリである。単に締切日や予定を並べるのではなく、日々の予定で使えない時間・生活時間・タスクの見積もり時間・予定が未確定な期間の見込み時間を踏まえて、タスクごとの猶予時間を表示する。

これにより、ユーザーは「どのタスクに早く着手すべきか」「今の予定量で締切に間に合うか」を判断できる。

## 2. 用語定義

| 用語                                                                | 定義                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 予定（スケジュール / Schedule）                                     | 開始・終了時刻が決まっている時間ブロック。会議、授業、外出、睡眠などの生活時間も含む。カレンダー上に時間占有として表示される。（コメント：睡眠とかの時間は1日に使える時間の設定で考慮する。睡眠が長ければ1日に使える時間を短くするみたいに） |
| タスク（Task）                                                      | 締切日と所要時間（見積もり）を持つ作業項目。締切日のセルに表示され、猶予時間計算の対象となる。（コメント：google calendarのタスクのデータ構造に対応したいから、要詳細化）                                                                    |
| 一時的なタスク（TempTask）                                          | 締切・所要時間が未定の用事を名前だけで素早く記録する軽量メモ。後から本タスクへ変換できる。                                                                                                                                                   |
| 猶予時間（残り時間 / Remaining Time）                               | あるタスクの締切までに、予定・生活時間・他タスクの所要時間・未確定期間の見込みを差し引いた後に残る、実際に作業へ使える時間。負値は時間不足を意味する。                                                                                       |
| 1日に使える時間（availableTimePerDay）                              | 睡眠等の生活時間を除き、1日のうち作業に充てられる時間の上限（時間単位）。ユーザー設定値。                                                                                                                                                    |
| 予定確定最終日（dateTakeIntoAccount）                               | この日までは予定が確定済みとして扱う最終日。これ以降は予定が未入力でも見込みで計算する。ユーザー設定値。                                                                                                                                     |
| 予定未確定日の1日あたり使える時間（availableTimePerUnscheduledDay） | 予定確定最終日より先の未確定期間について、1日あたり作業に使えると見込む時間（時間単位）。ユーザー設定値。                                                                                                                                    |
| サボってもいい時間                                                  | 現在のタスク状況から算出される、1日平均で作業しなくても締切に間に合う余裕時間。                                                                                                                                                              |
| Google Calendar 同期                                                | Google Calendar の予定をアプリの予定として手動で取り込み、双方向に反映する機能。タスク・一時タスクは同期対象外。                                                                                                                             |

## 3. 解決したいユーザー課題

- 期限が遠いタスクを後回しにしてしまい、締切直前まで危機感を持ちにくい。
- カレンダー上の予定とToDoリストが分かれているため、実際に作業へ使える時間が見えにくい。
- タスクの所要時間を考慮したとき、締切までに余裕があるのか不足しているのかを直感的に判断しにくい。
- 思いついた小さな用事をすぐ記録したいが、締切や所要時間まで決まっていない段階では正式なタスクにしづらい。
- 普段使っている Google Calendar の予定も、タスクの猶予時間計算に反映したい。

---

## 4. 想定ユーザー

- 個人の予定とタスクをまとめて管理したいユーザー。
- 学習、仕事、制作、事務作業など、締切と見積もり時間があるタスクを複数抱えるユーザー。
- 予定の空き時間ではなく、締切までに実際に使える作業時間を確認したいユーザー。
- タスク管理を義務感だけでなく、「時間を貯金する」感覚で前向きに続けたいユーザー。

利用形態は **個人利用** を主な前提とし、チーム共有・組織管理は本リリースのスコープ外とする（[11. 制約・前提](#11-制約前提)）。（コメント：OAuth2.0の勉強をしたくて、spring securityとかを使ってrole管理して、roleの違いとかで編集権限とか見れる内容とかを変えてみたいので、チーム共有・組織管理とかは入れたいなと思っています。PKCEとかも有効化してみたい）

## 5. スコープ

### 5.1 本リリース（MVP）に含むも

参考リポジトリで実装・検証済みの全機能を、本リリースのスコープとする。

- カレンダー表示（月単位）
- 予定の追加・編集・削除（繰り返し、複数日対応含む）
- タスクの追加・編集・削除
- 猶予時間の算出・可視化（「サボってもいい時間」表示を含む）
- 一時的なタスクの記録・削除・本タスクへの変換
- 設定（1日に使える時間、予定確定最終日、予定未確定日の1日あたり使える時間）
- Google アカウントによるログイン／ログアウト
- Google Calendar との手動同期（予定のみ）

### 5.2 本リリースに含まないもの（将来構想）

[12. 将来構想](#12-将来構想) に記載の機能（プロジェクト管理、タスク完了とモチベーション、モバイル最適化、AI支援、チーム・外部サービス連携）は本リリースの対象外とする。ただし、データモデル・アーキテクチャはこれらの拡張を阻害しない構造を目指す。

## 6. 機能要件

要件IDは `FR-<領域>-<連番>` 形式で付与する。

### 6.1 カレンダー表示（FR-CAL）（コメント：カレンダー表示は別の参考のレポのものと同じものにするつもりはなく、もっと使いやすい表示を考えて柔軟に変えていきたい）

| ID        | 要件                                                                                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-CAL-01 | ユーザーは月単位のカレンダーで予定とタスクを確認できる。（コメント：月単位以外も表示できる様にしたい。google calendarのようなUIもありだなと思っている） |
| FR-CAL-02 | ユーザーは前月・翌月・今日へ移動できる。                                                                                                                |
| FR-CAL-03 | カレンダーの日付セルには、その日の予定名と、締切日が一致するタスク名が表示される。                                                                      |
| FR-CAL-04 | 選択中の日付・今日・土曜日・日曜日・祝日が視覚的に区別される。                                                                                          |
| FR-CAL-05 | 日付を選択すると、その日の予定とタスクの詳細を同じ画面で確認できる。                                                                                    |
| FR-CAL-06 | 予定は青系、タスクは緑系で表示し、種類を区別しやすくする。（コメント：別にこの要件はいらない）                                                          |
| FR-CAL-07 | 複数日にまたがる予定は、該当する各日のセルに表示される。                                                                                                |

補足: 祝日判定は日本の祝日を対象とする（参考実装では `japanese-holidays` 相当の判定を使用）。

### 6.2 予定管理（FR-SCH）

| ID        | 要件                                                                                                                                                                                      |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-SCH-01 | ユーザーは予定を追加・編集・削除できる。                                                                                                                                                  |
| FR-SCH-02 | 予定には、予定名・開始時刻・終了時刻・繰り返し・場所・メモを登録できる。（コメント：google calendarの同期ができるようにデータ構造揃えたい。だから登録できる項目はその要件によると思う。） |
| FR-SCH-03 | 予定名・開始時刻・終了時刻は必須とする。                                                                                                                                                  |
| FR-SCH-04 | 開始時刻が終了時刻より後の予定は登録できない（バリデーション）。                                                                                                                          |
| FR-SCH-05 | 繰り返し予定は「なし・毎日・毎週・毎月・毎年」から選べる。（コメント：google calendarの同期ができるようにgoogle calendarの繰り返しに合わせたい）                                          |
| FR-SCH-06 | 繰り返し予定には繰り返し開始日・繰り返し終了日（無期限可）を持たせる。（コメント：これもgoogle calendarにあるんかな？調べて欲しい）                                                       |
| FR-SCH-07 | 繰り返し予定の編集・削除では、対象を「この予定のみ／以降すべて／すべて」など適切な単位で選べる。（コメント：これもgoogle calendarに合わせたい）                                           |
| FR-SCH-08 | 複数日にまたがる予定を登録できる。                                                                                                                                                        |

補足: FR-SCH-07 の選択肢の正確な仕様は設計フェーズで確定する（参考実装に繰り返し編集オプション用UIあり）。除外日（特定回のみスキップ）を保持できる構造とする。

### 6.3 タスク管理（FR-TASK）

| ID         | 要件                                                                                                                                                                                                                                                        |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-TASK-01 | ユーザーはタスクを追加・編集・削除できる。                                                                                                                                                                                                                  |
| FR-TASK-02 | タスクには、タスク名・締切日・所要時間・メモを登録できる。（コメント：google calendarの同期ができるようにデータ構造揃えたい。だから登録できる項目はその要件によると思う。ただ、所要時間はなかった気がするからこれはこっち独自の項目として持っておきたい。） |
| FR-TASK-03 | タスク名・締切日・所要時間は必須とする。                                                                                                                                                                                                                    |
| FR-TASK-04 | 所要時間は正の数（時間単位）で入力する。                                                                                                                                                                                                                    |
| FR-TASK-05 | タスクは締切日のセルにカレンダー上で表示される。                                                                                                                                                                                                            |
| FR-TASK-06 | タスクリストでは、タスク名・締切日・所要時間・猶予時間を確認できる。                                                                                                                                                                                        |
| FR-TASK-07 | ユーザーはタスクリストを列ごとに並び替えられる。（コメント：猶予時間順で並び替えたりってことね。自由に並び替えれるわけではないですよ）                                                                                                                      |

補足: 本リリースではタスクの完了状態を明示管理しない。作業対象から外す場合は編集・削除で対応する（将来構想でタスク完了を追加予定。[12.2](#122-タスク完了とモチベーション)）。

### 6.4 猶予時間の可視化（FR-REM）

| ID        | 要件                                                                                                                                                                                                                                |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-REM-01 | ユーザーはタスクごとに、締切までの猶予時間を確認できる。                                                                                                                                                                            |
| FR-REM-02 | 猶予時間は、1日に使える時間・予定で使う時間・締切日までに存在する他タスクの所要時間・未確定期間の見込みを踏まえて算出する（詳細は[9. 猶予時間計算仕様](#9-猶予時間計算仕様)）。                                                        |
| FR-REM-03 | 予定で使う時間は、1日に使える時間を上限として差し引く。                                                                                                                                                                             |
| FR-REM-04 | 予定が未確定の将来期間については、設定された「予定未確定日の1日あたり使える時間」を使って見込みを調整する。（コメント：これって予定未確定日を設定しないという動線を用意して、無限に予定確定日とするモードも欲しいなと思いました。） |
| FR-REM-05 | 猶予時間がマイナスの場合は、締切までに時間が不足している状態として表示する。                                                                                                                                                        |
| FR-REM-06 | 猶予時間が少ないタスク、およびそれより締切が早いタスクは、目立つように表示し優先度に気づけるようにする。                                                                                                                            |
| FR-REM-07 | 画面上部に、現在のタスク状況から見た「1日平均でサボってもいい時間」を表示する。（コメント：別に丈夫じゃなくてもいい）                                                                                                               |

### 6.5 一時的なタスク（FR-TMP）

| ID        | 要件                                                                               |
| --------- | ---------------------------------------------------------------------------------- |
| FR-TMP-01 | ユーザーは、締切や所要時間が未定の用事を一時的なタスクとして記録できる。           |
| FR-TMP-02 | 一時的なタスクは名前だけで追加できる。                                             |
| FR-TMP-03 | ユーザーは一時的なタスクを削除できる。                                             |
| FR-TMP-04 | ユーザーは一時的なタスクに締切日・所要時間・メモを追加して、本タスクへ変換できる。 |
| FR-TMP-05 | 本タスクへ変換された一時的なタスクは、一時的なタスク一覧から削除される。           |

### 6.6 設定・認証（FR-SET）

| ID        | 要件                                                                                                                                                 |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-SET-01 | ユーザーは Google アカウントでログイン・ログアウトできる。（コメント：別にgoogleだけに限らなくてもいい）                                             |
| FR-SET-02 | ログインしていない場合、予定・タスク・一時タスク・設定の保存操作は行えない。（コメント：ログインしていない場合はログイン画面に行く様にしたらどうか） |
| FR-SET-03 | ユーザーは「1日に使える時間」を設定できる。                                                                                                          |
| FR-SET-04 | ユーザーは「予定を確定済みとして扱う最終日（予定確定最終日）」を設定できる。                                                                         |
| FR-SET-05 | ユーザーは「予定未確定日の1日あたり使える時間」を設定できる。                                                                                        |
| FR-SET-06 | 各時間設定は正の数で入力する（バリデーション）。                                                                                                     |
| FR-SET-07 | 設定はユーザーごとに永続化され、再ログイン後も保持される。                                                                                           |

### 6.7 Google Calendar 同期（FR-GC）

| ID       | 要件                                                                                                                                         |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-GC-01 | ユーザーは Google Calendar の予定をアプリの予定として同期できる。                                                                            |
| FR-GC-02 | 同期対象は予定のみとし、タスク・一時タスクは Google Calendar へ同期しない。                                                                  |
| FR-GC-03 | 同期は自動ではなく、ユーザーが設定画面から「今すぐ同期」を実行したときに行う。                                                               |

| FR-GC-06 | アプリで作成した予定は、同期後に Google Calendar 側にも作成される。                                                                          |
| FR-GC-07 | Google Calendar 側で作成した予定は、同期後にアプリ側にも表示される。                                                                         |
| FR-GC-08 | 同期済み予定の更新・削除は、同期後に双方へ反映される。                                                                                       |
| FR-GC-09 | アプリ側と Google Calendar 側の両方で同じ予定を変更した場合は、Google Calendar 側の内容を優先する。                                          |
| FR-GC-10 | 同期結果として、取込件数・更新件数・作成件数・削除件数・競合解決件数・エラー件数をユーザーへ表示する。                                       |
| FR-GC-11 | 複雑な繰り返し予定・終日予定・参加者付き予定・会議リンク付き予定・特殊な Google 予定は、表示できてもアプリ上で完全に編集できるとは限らない。 |
| FR-GC-12 | 未対応の Google 繰り返し予定は、ユーザーが警告に気づけるように表示する。                                                                     |

## 8. データモデル（論理）

実装技術に依存しない論理モデルとして定義する。物理スキーマ（テーブル/コレクション設計）は設計フェーズで確定する。データはユーザー単位で分離する。

| location          | 文字列   | −   | 場所                                     |
| memo              | 文字列   | −   | メモ                                     |

| ------------------------------ | ------------ | ---- | ------------------------------------------------------------------------------------ |
| availableTimePerDay            | 数値         | ○   | 1日に使える時間（時間単位、正の数）                                                  |
| dateTakeIntoAccount            | 日付         | ○   | 予定確定最終日                                                                       |
| availableTimePerUnscheduledDay | 数値         | ○   | 予定未確定日の1日あたり使える時間（時間単位、正の数）                                |

## 9. 猶予時間計算仕様

本アプリの中核ロジック。参考実装（`RemainimgTimeUtil.ts`）で検証済みの定義を踏襲する。すべての時間は「時間（hour）」単位、日付計算は日単位で行う。

### 9.1 日別予定時間の事前計算

今日から計算最終日までの各日について、その日に予定が占有する合計時間を求める。

1. タスクのうち最も遅い締切日を求め、それと `dateTakeIntoAccount` の **早い方** を計算最終日とする。
2. 今日から計算最終日まで1日ずつ進めながら、各日に該当する予定（繰り返し展開・複数日跨ぎを含む）の占有時間を合算する。
   - 繰り返し予定は種類（daily/weekly/monthly/yearly）に応じて当該日に発生するかを判定し、跨ぎ日はその日の 0:00〜23:59 にクリップして時間を算出する。
3. 各日の合計予定時間は **`availableTimePerDay` を上限** とする（予定が1日の作業可能時間を超えても、差し引くのは作業可能時間まで）。

結果は「日付 → その日の予定占有時間（上限あり）」のマップとして保持する。

### 9.2 タスクごとの猶予時間

対象タスクについて、以下を順に計算する。

1. **締切までの日数** = 締切日 − 今日（日単位）。
2. **総利用可能時間（予定考慮前）** = 締切までの日数 × `availableTimePerDay`。
3. **予定で使う時間の差し引き**: 今日から「締切」と「`dateTakeIntoAccount`」の早い方までの各日について、9.1 のマップ（上限適用済み）の予定時間を合算し、総利用可能時間から差し引く。
4. **他タスク所要時間の差し引き**: 対象タスクを含む、締切日が対象タスクの締切日 **以前（同日含む）** のすべてのタスクの所要時間合計を差し引く。
5. **未確定期間の補正**: 締切が `dateTakeIntoAccount` より先にある場合、その未確定期間の各日について `(availableTimePerDay − availableTimePerUnscheduledDay)` を差し引く（未確定日は確定日より作業見込みを下げる）。

結果が **正** なら締切まで余裕あり、**負** なら時間不足を示す。

### 9.3 サボってもいい時間

現在のタスク状況から、1日平均で作業しなくても締切に間に合う余裕時間を算出し、画面上部に表示する（算出式の正確な定義は参考実装に準拠し、設計フェーズで明文化する）。

### 9.4 表示ルール

- 猶予時間が少ないタスク、およびそれより締切が早いタスクは強調表示する（FR-REM-06）。
- 猶予時間が負のタスクは時間不足として明示する（FR-REM-05）。

> 注: 本章は参考実装のロジックを要件として固定するものであり、移植時の回帰を防ぐため、これら計算は純粋関数として実装し単体テストで保護する（[10.4](#104-品質テスト方針)）。

draft の理想方針を基本とする。各方針は、本アプリに不適切と判断された場合、より適切な選択肢へ変更しうる。具体的な製品・サービス選定（認証基盤、データストア等）は設計フェーズで確定する（本書は方式非依存）。

### 10.1 技術スタック

| 層                 | 採用方針                                     |
| ------------------ | -------------------------------------------- |
| バックエンド言語   | Java                                         |

- バックエンド: **オニオンアーキテクチャ**（ドメイン中心、依存方向を内向きに保つ）。猶予時間計算等のコアロジックはドメイン層の純粋ロジックとして配置する。
- フロントエンド: **FSD（Feature-Sliced Design）** を基本とする。
- いずれも本アプリに不適切と判断した場合は、より適切なアーキテクチャを採用しうる。

### 10.3 認証・データ永続化（方式非依存）

本書では方式に依存しない要件のみ定義する。具体製品は設計フェーズで決定する。

- ユーザーは Google アカウントでログインできること（FR-SET-01）。
- データはユーザー単位で分離・永続化され、再ログイン後も保持されること（FR-SET-07）。

### 10.4 品質・テスト方針

- 参考実装の既知の挙動を回帰テストの基準とする。

## 11. 制約・前提

- 本リリースは個人利用を主な前提とし、チーム共有・組織管理は対象外とする。
- タスク・一時タスクは Google Calendar と同期しないため、タスク管理はアプリ内で完結する。
- Google Calendar 同期は、最初は専用のテストカレンダーで試すことを推奨する。
- 複雑な繰り返し予定・終日予定・参加者付き会議・特殊な Google 予定は、Google Calendar 側で編集する運用を推奨する。
- 同期範囲外の予定は標準設定では取り込まれない。同期範囲内に大量の予定がある場合、すべてを取り込めない可能性がある。
- 本リリースではタスクの完了状態を明示管理しない（編集・削除で対応）。
- 個人開発・趣味の範囲のため、可用性・SLA 等の厳密な保証は本リリースの対象外とする。

## 12. 将来構想

本リリースのスコープ外。データモデル・アーキテクチャはこれらの拡張を阻害しない構造を目指す。

### 12.1 プロジェクト管理

- 複数のタスクをプロジェクト単位でまとめられる。
- 大きなタスクを小さなタスクへ分割できる（入れ子構造）。
- プロジェクトごとに合計所要時間や進行状況を確認できる。
- 既存タスクをプロジェクトへ紐づけられる。

### 12.2 タスク完了とモチベーション

- タスクを完了済みとして記録できる。
- タスクを早めに進めると猶予時間が増える感覚を得られる。
- 着手の遅れや未完了タスクの増加により猶予時間が減ることを確認できる。
- 「時間を貯金する」体験として、継続的な取り組みを促す。

### 12.3 モバイル対応

- スマートフォンからも予定・タスク・猶予時間を確認できる。
- 外出先でも一時的なタスクをすばやく登録できる。
- モバイル環境でもカレンダーとタスクリストの主要操作を無理なく行える。

### 12.4 AI支援

- タスクの所要時間や優先度の判断を支援する。
- タスク量と予定状況から、着手すべきタスクを提案する。
- 締切までに時間が不足しそうな場合、計画の見直し案を提示する。

### 12.5 チーム・外部サービス連携

- 複数人でのタスク共有や進捗確認に対応する。
- Slack など、Google Calendar 以外のサービスとの連携を検討する。
- 組織利用に必要な権限管理・共有範囲・監査性を拡張要件として扱う。
